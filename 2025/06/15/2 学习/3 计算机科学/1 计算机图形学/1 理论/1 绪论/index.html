图像显示原理

图形管线

  • 概念: 一个将抽象几何数据转换为最终屏幕像素的流水线处理过程。

  • 核心: 这是一个从逻辑空间到物理像素的、高度并行的转换系统。

图像表示

顶点属性

  • 位置 \((x,y,z)\)
  • 颜色 \((r,g,b,a)\)
  • 纹理坐标 \((u,v)\)
  • 法线 \((n_x,n_y,n_z)\)

多边形表示——三角形拼接

基于性能的考虑，OpenGL核心模式中不存在像 glDrawPolygon() 这样直接绘制任意多边形（四边形、五边形等）的单一命令。

标准的、最基本且最通用的方法就是用三角形来拼接绘制多边形

为什么颜色属于顶点，而非面？

核心原因是为了通过插值(Interpolation)实现平滑的视觉效果。

  • 灵活性: 顶点系统可以通过“将三顶点设为同色”来实现平面着色。反之则不行。
  • 真实光照: 光照计算在顶点上进行，其结果（明暗）可通过插值形成平滑过渡，模拟真实曲面。
  • 硬件高效: GPU硬件天生为高速执行顶点属性插值而优化。
对比项 颜色属于顶点 (主流/平滑着色) 颜色属于面 (限制/平面着色)
原理 GPU自动对顶点间的颜色进行插值，计算出三角形内部每个像素的颜色。 整个三角形填充为单一颜色。
效果 颜色平滑渐变，隐藏了三角形边界，使模型看起来更真实、圆润。 棱角分明，呈块状感，适用于Low Poly等特定艺术风格。

下面是一个空间中红色四边形的定义，用两个三角形拼接而成

float vertices[] = {
    // 位置              // 颜色
    -0.5f, -0.5f, -0.5f,  1.0f, 0.0f, 0.0f, // 红
    0.5f, -0.5f, -0.5f,  1.0f, 0.0f, 0.0f,
    0.5f,  0.5f, -0.5f,  1.0f, 0.0f, 0.0f,
    0.5f,  0.5f, -0.5f,  1.0f, 0.0f, 0.0f,
    -0.5f,  0.5f, -0.5f,  1.0f, 0.0f, 0.0f,
    -0.5f, -0.5f, -0.5f,  1.0f, 0.0f, 0.0f,
}

着色器

  • 概念: 在图形处理单元（GPU）上运行的、用户定义的小型程序。
  • 职责:
    • 顶点着色器 .vert: 定义几何体顶点的最终位置。(输入: 顶点 -> 输出: 裁剪空间坐标)
    • 片元着色器 .frag: 定义屏幕上每个像素片元的最终颜色。(输入: 表面属性 -> 输出: 颜色值)
  • 核心: 赋予开发者在渲染管线的关键阶段（几何变换与着色）进行编程控制的能力。

坐标系变换

  • 概念: 通过矩阵乘法，将顶点从一个坐标空间变换到另一个坐标空间的过程。
  • 核心: 建立一套数学框架，用于在二维屏幕上模拟三维世界的布局与观察。

缓冲对象与状态管理

  • 概念: 一种数据管理策略，用于在CPU和GPU之间高效地传输和访问数据。
  • 机制:
    • 缓冲对象 (e.g., VBO): 在GPU上分配专用内存，用于存储几何体、纹理等大数据块。
    • 状态对象 (e.g., VAO): 封装渲染所需的所有状态配置（如数据格式、缓冲绑定），以简化渲染调用。
  • 核心: 将数据与指令分离，最小化CPU到GPU的通信开销，实现高性能渲染。

渲染循环

  • 概念: 实时图形应用的心跳，一个持续执行的无限循环。
  • 基本步骤:
    1. 输入: 处理用户交互。
    2. 更新: 更新场景状态（如物体位置、动画）。
    3. 渲染: 执行绘图指令，生成一帧图像。
  • 核心: 通过以高频率（如60 FPS）重复生成静态图像，创造出连续运动的视觉效果。
