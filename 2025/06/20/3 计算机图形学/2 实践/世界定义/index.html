新建 world 类

主要职责：

  • 作为一个世界，管理世界中的对象
  • 渲染所管理的对象
class World {
public:
    World() = default;
    ~World() = default;

    //对比 void AddObject(Object* object);
    void AddObject(std::unique_ptr<Object> object);
    void Render(const glm::mat4& view, const glm::mat4& projection);

private:
    // 主列表，负责对象的所有权
    std::vector<std::unique_ptr<Object>> m_Objects;

    // 渲染批处理列表，用于高效渲染
    // Key: Shader指针, Value: 使用该Shader的Object裸指针列表
    std::map<Shader*, std::vector<Object*>> m_RenderBatches;
};

世界渲染

此处给着色器 shader 传入视图矩阵、投影矩阵。

模型变换矩阵（从模型局部坐标到世界坐标变换）由对象自身 obj->Render() 进一步提供，实现渲染。

提升渲染速度策略：降低渲染管线中着色器切换开销，先渲染完同一种着色器上色对象再切换另一种。

使用 map 来管理，key就是着色器指针，value就是对象数组。

void World::AddObject(std::unique_ptr<Object> object) {
    // 获取对象的着色器
    Shader *shader = object->GetShader();
    // 将对象的裸指针添加到对应着色器的批处理列表中
    m_RenderBatches[shader].push_back(object.get());

    // 将对象的所有权转移到主列表中
    m_Objects.push_back(std::move(object));
}

void World::Render(const glm::mat4 &view, const glm::mat4 &projection) {
    // 遍历所有渲染批次
    for (auto const &[shader, objects]: m_RenderBatches) {

        // 1. 激活着色器 (每个批次只调用一次!)
        shader->use();

        // 2. 设置该批次共享的Uniforms (如视图和投影矩阵)
        shader->setMat4("view", view);
        shader->setMat4("projection", projection);
        // 如果有其他共享的uniform (如光照参数)，也在这里设置

        // 3. 遍历并渲染该批次中的所有对象
        for (Object *obj: objects) {
            // 注意：现在由 World 来调用 obj->Render
            // Object::Render 内部只设置 model 矩阵和执行绘制
            obj->Render();
        }
    }
}
