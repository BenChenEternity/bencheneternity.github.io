什么是GNU？

递归缩写 “GNU’s Not Unix”，强调其与 Unix 的哲学区别。

GNU 的核心目标是创建一个完全由自由软件组成的类 Unix (Unix-like) 操作系统。

它强调用户有权运行、复制、分发、学习、修改和改进软件。

什么是 GCC?

GCC (GNU Compiler Collection) 是目前最主流、最重要的编译器套件之一。用于编译源代码为机器码，包括 C、C++、Objective-C、Fortran、Ada、Go 等语言。

特点：

  • 开源
  • 跨平台 - Linux Windows MacOS …

传统意义 GCC 是纯编译器，不含链接器和汇编器。

实际使用中，输入 gcc main.c -o main 进行编译，实际上此处的 gcc 是一个启动器，负责启动构建流程：预处理-编译-汇编-链接。

什么是 CMake？

CMake 是一个开源、跨平台的自动化构建系统。用于管理和简化软件的编译构建过程，尤其适用于大型、复杂的项目。默认使用 gcc 进行编译。

通过解析 CMakeLists.txt 文件构建编译任务。

什么是 MinGW？

“Minimalist GNU for Windows” 的缩写。它是一个开源的 C++ 编译器和工具链，是 GCC 在 Windows 平台上的移植版。gcc 在 windows 平台不兼容，必须使用 MinGW。

包含 GCC

作用也是编译。

什么是 MSVC？

MSVC (Microsoft Visual C++) 是微软公司开发的 C、C++ 和 C++/CLI 编译器。与 Windows 操作系统、Windows API 以及其他微软开发技术（如 .NET, DirectX）的集成度是最高的。

如何选择 MinGW/GCC 和 MSVC？

MSVC 和 GCC (MinGW) 编译的二进制文件（.obj, .lib, .dll）不能混用。你必须为你的整个项目选择一个统一的编译器工具链。例如，如果你需要链接一个只提供了 MSVC 版本的 .lib 库，那么你的整个项目也必须用 MSVC 来编译。

“我要为 Windows 开发一个原生程序，特别是带界面的那种，并且希望有最强大的图形化调试工具。” -> 选择 Visual Studio 和 MSVC。这是最顺畅、最高效的路径。

“我写了一个程序，希望它既能在我的 Windows 电脑上运行，也能在同学的 Linux 服务器上无缝编译运行。” -> 选择使用 MinGW-w64 (GCC) 和跨平台构建工具 CMake。

“我需要在 Windows 上编译一个从 GitHub 下载的开源项目。” -> 查看它的编译指南。如果它使用 Makefile 或 Autotools，那么你几乎肯定需要 MinGW-w64 (GCC)。如果它提供的是 .sln 文件，那你需要 Visual Studio (MSVC)。

什么是 .NET？

.NET 是一个免费、开源的软件开发平台。它提供了一个全面的环境和一套工具，让开发者可以构建、部署和运行各种类型的应用程序，比如网站、桌面应用、云服务、移动应用和游戏等。

使用 .NET 开发的应用程序可以一次编写，在 Windows、Linux 和 macOS 上原生运行。主要使用 C#。
