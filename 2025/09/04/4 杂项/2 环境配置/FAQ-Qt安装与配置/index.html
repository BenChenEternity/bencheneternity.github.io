FAQ-Qt安装与配置

online-installer 获取信息和下载特别慢？

使用国内镜像源下载，具体方式：在终端中启动并且加入启动参数，例如

C:\path\to\qt\qt-online-installer-windows-x64-4.10.0.exe --mirror https://mirrors.ustc.edu.cn/qtproject

安装 Qt 各种组件的时候，应该如何选择？

基本的配置是这样

  • Qt
    • MinGW
    • Qt Quick Timeline
    • Additional Libraries
  • Qt Design Studio
  • Build Tools
    • MinGW
    • Qt Installer Framework
    • CMake
    • Ninja

注意：

在 Qt 的 MinGW、Build Tools 里的、独立安装的 MinGW 的区别是什么？

  • Qt 的 MinGW 是指用 MinGW 编译后的 Qt 库。
  • Build Tools 是 MinGW 编译器本身。

Build Tools 中的 MinGW 是一个“特供版”，它经过 Qt 官方的测试和配置，保证与该版本的 Qt 无缝协作；而自己独立安装的 MinGW 是一个“通用版”，版本可能更新，但需要手动配置才能与 Qt 正常工作。

Additional Libraries 有什么用

模块 (Module) 一句话用途
Qt Charts 用于绘制折线图、柱状图、饼图等各种二维数据图表。
Qt Data Visualization 用于将数据显示为可交互的3D柱状图、散点图或曲面图。
Qt Quick 3D 用于在QML界面中轻松集成和渲染3D模型与场景。
Qt 5 Core Compatibility Module 为老旧的Qt 5项目迁移到Qt 6提供API兼容性支持。
Qt WebEngine 在你的应用程序中嵌入一个功能完整的Chromium浏览器内核。
Qt WebChannel 实现C++/QML后端代码与嵌入网页中的JavaScript进行双向通信。
Qt Network Authorization 提供对Google、Facebook等网络服务的OAuth 1/2登录认证功能。
Qt Multimedia 提供音视频播放、录制以及调用摄像头等核心多媒体功能。
Qt Serial Port 用于通过串口（COM口）与Arduino、单片机等硬件设备进行通信。
Qt Bluetooth 用于发现蓝牙设备并与其进行数据通信。
Qt NFC 提供近场通信（NFC）功能，用于读取NFC标签或设备间数据交换。
Qt PDF 让你的应用程序能够渲染显示或生成PDF格式的文档。
Qt SVG 使你的应用程序能够加载和渲染SVG格式的矢量图形。

Qt Design Studio 和 Qt Creator 的区别

特性 Qt Design Studio Qt Creator 内的 UI 设计器
目标用户 UI/UX 设计师 程序员
核心焦点 视觉表现、动画、交互、原型制作 功能实现、快速布局、信号与槽连接
动画能力 非常强大 (时间轴、关键帧) 非常有限 (主要通过属性绑定)
3D 支持 原生集成，可进行 3D 场景编辑 基本不支持，需要手写 QML 代码
设计导入 支持 (Figma, Photoshop 等) 不支持
最终产出 高质量的 QML 代码 (定义了界面的外观和感觉) QML 或 .ui 文件 (定义了界面的结构和基本属性)

CDB Debugger Support 和 Debugging Tools for Windows 是做什么的？

CDB Debugger Support 是一个让 Qt Creator 能够在 Windows 系统上、在使用微软 MSVC 编译器时，具备代码调试功能的核心组件。

没有它，在 Windows 上用 MSVC 编译代码时，Qt Creator 里的“调试”按钮就等于废了。

  • CDB (MSVC)：CDB 是 Windows 操作系统的一部分，由微软通过 “Debugging Tools for Windows” 这个独立的包来分发。Qt 本身不包含它，所以 Qt 安装器需要一个额外的选项来提醒你，并帮你从微软那里下载安装。

  • GDB (GCC)：GDB 和 GCC 编译器本身就是一家人，它们通常被打包在一起。当你下载 MinGW 工具链时，里面不仅包含了编译器 g++.exe，通常也已经包含了调试器 gdb.exe。

如何在 Clion 中配置 Qt 开发环境？

设置 - 工具链

配置工具链为 qt 下载的 MinGW

C:\path\to\qt\Tools\mingw1310_64\bin

然后在 Qt CMake 前缀路径设置为

C:\path\to\qt\6.7.3\mingw_64

什么是 Ninja？

Ninja 是一个专注于极致速度的小型、现代化的构建系统。它本身不负责规划项目如何编译的复杂逻辑，而是像一个高效的“施工队”，专门负责执行由 CMake 这样的“项目经理”所生成的简单、明确的编译指令。其核心优势在于极低的开销和出色的并行处理能力，能够充分利用多核CPU，从而显著缩短大型项目的编译等待时间，尤其是在只修改了少量文件的增量构建中表现卓越。因此，Ninja 通常与 CMake 搭档使用，作为其后端，共同构成了一套高效、快速的现代C++开发构建方案。
